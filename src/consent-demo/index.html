Security Research Demo

Educational demonstration of common web security vulnerabilities.

Geolocation Access

Shows how websites can track your location.

Prevention:

  • Use browser location permissions carefully
  • Enable location only for trusted sites
  • Use VPN to mask real location

Keylogging Detection

Demonstrates keystroke monitoring.

Prevention:

  • Use virtual keyboards for sensitive data
  • Install anti-keylogger software
  • Keep system updated against malware

XSS Prevention

Shows how malicious scripts can be injected.

Prevention:

  • Always sanitize user input
  • Use Content Security Policy (CSP)
  • Enable HTTPOnly cookies
  • Implement input validation
  • Use modern framework escaping

Try these XSS examples:

<script>alert('xss')</script> <img src="x" onerror="alert('xss')"> <a href="javascript:alert('xss')">click me</a> <svg onload="alert('xss')"></svg>

Browser Fingerprint

Shows unique browser identification data.

Prevention:

  • Use privacy-focused browsers
  • Enable fingerprinting protection
  • Use browser extensions like Privacy Badger

CSRF Attack

Shows how cross-site request forgery works.

Prevention:

  • Implement CSRF tokens
  • Check Origin and Referer headers
  • Use SameSite cookie attribute
  • Require re-authentication for sensitive actions

Example CSRF attack:

<img src="http://bank.com/transfer?amount=1000&to=attacker">

SQL Injection

Demonstrates SQL injection vulnerabilities.

Prevention:

  • Use parameterized queries
  • Input validation and sanitization
  • Principle of least privilege
  • Regular security audits

Try these SQL injection examples:

admin' OR '1'='1 '; DROP TABLE users; -- ' UNION SELECT * FROM passwords --

Local Storage Security

Shows risks of storing sensitive data.

Prevention:

  • Never store sensitive data in localStorage
  • Use HttpOnly cookies for auth tokens
  • Implement proper session management
  • Encrypt sensitive client-side data

⚠️ Educational purposes only. No data is stored or transmitted.
